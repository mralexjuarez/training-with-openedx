What and Why of Open edX

edX.org

We can not talk about Open edX without talking about edx.org first.

edX.org is the online learning destination founded by Harvard and MIT in 2012

https://open.edx.org/about-open-edx/

Open edX?

Open edX is the platform is made up of a LMS and a CMS

  • LMS (Open edX) - Learning Management System
  • CMS (Studio X) - Content Management System

Employee Onboarding

The year is 2017 and we are revamping / ramping up our onboarding process.

We had a solution, and it was only after using it that we decided we wanted something else.

Our Objectives

  • Have a consistent onboarding process
  • Be able to offer a self-service solution
  • Courses need to be easy to modify / update.

Open edX Benifets (for us)

  • Easy to Try it Out
  • Easy Course Authoring
  • Self Paced Courses
  • Third-Party Intergration

Updating Nginx

By default the Studio software runs on 18010

Simply changing the listen directive to 80 will allow us to run on a sub-domain of our primary domain.

Configuring Sites

{
  "course_email_from_addr":"courses@online.unsupported.io",
  "university":"Online University",
  "PLATFORM_NAME":"Online Demo Program",
  "email_from_address":"courses@online.unsupported.io",
  "SITE_NAME":"online.unsupported.io",
  "site_domain":"online.unsupported.io",
  "SESSION_COOKIE_DOMAIN":"online.unsupported.io",
  "platform_name":"online.unsupported.io"
}

Restarting Services

  • LMS - sudo /edx/bin/supervisorctl restart lms
  • CMS - sudo /edx/bin/supervisorctl restart cms
  • Workers - sudo /edx/bin/supervisorctl restart edxapp_worker:
